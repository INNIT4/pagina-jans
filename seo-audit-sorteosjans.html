🚨 Issues Críticos — Corregir inmediatamente

Crítico Contenido Datos de prueba en producción en /rifas
El listing de rifas activas muestra una rifa con nombre: "1" y descripcion: "1", y otra con descripcion: "123". Cualquier crawler o quality rater que visite el sitio verá esto. Activa los clasificadores de spam de Google y destruye la confianza de usuarios reales.
Fix: Eliminar o completar estas entradas desde el panel admin de Firestore. 5 minutos.
Crítico Performance ISR roto — cada request llega al origen (TTFB ~560ms)
La llamada a headers() en app/layout.tsx (para generar el nonce CSP) opta el árbol de rutas completo en dynamic rendering, sobreescribiendo export const revalidate = 60. Todas las páginas retornan Cache-Control: private, no-cache, no-store y X-Vercel-Cache: MISS en cada request. El TTFB debería ser ~50ms desde edge cache; actualmente es ~560ms desde origen para cada visita.
Fix: Mover la generación del nonce de layout.tsx a un Server Component que no llame a headers(), o leer el nonce desde request.headers en el middleware y pasarlo como response header, en vez de llamar headers() en el componente.
Crítico Schema Event schema sin endDate — Google rich results bloqueados
En app/rifas/[id]/page.tsx, el schema Event tiene tres errores: (1) falta endDate — requerido por Google para Event rich results, (2) price: String(rifa.precio_boleto) debe ser número no string, (3) falta seller en Event > offers.
Fix (3 cambios de una línea): 
endDate: rifa.fecha_sorteo,          // agregar
price: rifa.precio_boleto,           // quitar String()
seller: { "@id": `${BASE_URL}/#organization` }  // agregar en Event offers
Crítico Indexabilidad Redirect 307 Temporal en dominio apex — no transfiere PageRank
https://sorteosjans.com.mx → https://www.sorteosjans.com.mx usa HTTP 307 Temporary Redirect. Google no consolida link equity en redirects 307. Cualquier enlace externo o compartido en WhatsApp que use el dominio sin www no transfiere autoridad.
Fix: En Vercel Dashboard → Project → Domains, configurar el apex domain como redirect permanente (301) hacia www. También aplicable para /cuentas → /tarjetas que también usa 307.

⚠️ Prioridad Alta — Corregir esta semana

Alto og:url apunta al homepage en todas las sub-páginas
/rifas, /faq, /sobre-nosotros, /consulta retornan og:url = https://www.sorteosjans.com.mx y el og:title/description del homepage. Las tarjetas de WhatsApp/Facebook de cualquier sub-página muestran el contenido del homepage. También confunde a sistemas de IA que atribuyen el contenido a la URL incorrecta.
Fix: En cada página agregar openGraph: { url: "https://www.sorteosjans.com.mx/rifas", title: "...", description: "..." } al export de metadata.
Alto LCP en /rifas: primera imagen usa loading="lazy"
La primera card de rifa es el elemento LCP de la página /rifas pero tiene loading="lazy", retrasando su fetch hasta detección in-viewport. Esto probablemente resulta en LCP "Needs Improvement" o "Poor" en móvil.
Fix en app/rifas/page.tsx: 
<Image ... priority={index === 0} />
Alto sizes="100vw" en imágenes de grid — 3-4× datos innecesarios
Las imágenes de las cards de rifa (en grid de 3 columnas en desktop) usan sizes="100vw". El browser solicita imágenes de ~1920px para un contenedor de ~400px. Transfiere 3-4× más datos de imagen de los necesarios.
Fix: 
sizes="(min-width: 1024px) 33vw, (min-width: 768px) 50vw, 100vw"
Alto Organization schema sin sameAs — no hay disambiguation de entidad
El nodo Organization en app/layout.tsx no tiene array sameAs. Google no puede vincular la entidad de marca con perfiles sociales, lo que dificulta el Knowledge Panel.
Fix en app/layout.tsx: 
"sameAs": [
  "https://www.facebook.com/sorteosjansn",
  "https://www.instagram.com/sorteosjansn"
]
Alto Falta ItemList schema en /rifas — no hay carousel de rich results
La página listing /rifas no tiene ningún schema. Un ItemList con las rifas activas habilitaría el carrusel de eventos/productos de Google en resultados de búsqueda.
Fix en app/rifas/page.tsx: 
const itemListSchema = {
  "@context": "https://schema.org",
  "@type": "ItemList",
  name: "Rifas Activas — Sorteos Jans",
  url: `${BASE_URL}/rifas`,
  itemListElement: rifas.map((rifa, i) => ({
    "@type": "ListItem",
    position: i + 1,
    url: `${BASE_URL}/rifas/${rifa.id}`,
    name: rifa.nombre,
  })),
};
Alto /rifas-previas/[id] sin generateMetadata — título es template literal
app/rifas-previas/[id]/page.tsx no exporta generateMetadata. Los títulos en resultados de búsqueda muestran el string del template "%s | Sorteos Jans" sin substitución.
Fix: Agregar generateMetadata con fetch desde Firestore, siguiendo el mismo patrón de app/rifas/[id]/page.tsx.
Alto IndexNow no implementado — nuevas rifas tardan días en indexarse en Bing
Bing, Yandex y otros motores compatibles con IndexNow no reciben notificación cuando se activa una rifa nueva. Para un negocio donde los boletos tienen fecha límite, el retraso de indexación tiene impacto comercial directo.
Fix: (1) Generar un UUID como key. (2) Hospedar en /public/{key}.txt. (3) Declarar en app/robots.ts. (4) Hacer POST a api.indexnow.org/indexnow desde el admin panel al activar una rifa. 
POST https://api.indexnow.org/indexnow
{
  "host": "www.sorteosjans.com.mx",
  "key": "{tu-key}",
  "urlList": ["https://www.sorteosjans.com.mx/rifas/{id}"]
}
Alto Contenido delgado en / y /sobre-nosotros — sin identidad del operador
Homepage: ~280 palabras (floor recomendado: 500). /sobre-nosotros: ~220 palabras, sin nombre del operador, sin año de fundación, sin conteo de rifas completadas, sin estado/ciudad. Para un sitio de transacciones financieras, la ausencia de identidad verificable es crítica para E-E-A-T.
Fix: Agregar en /sobre-nosotros: estado de operación, año del primer sorteo, total de rifas completadas (consultable desde Firestore), y nombre o inicial del operador. Agregar sección "Ganadores Recientes" en homepage (datos ya existen en Firestore).

Medio No existe /llms.txt — GEO score: 38/100

# Sorteos Jans

Medio Respuestas del FAQ demasiado cortas para citación de IA
Las 8 respuestas del FAQ miden entre 14 y 30 palabras. El rango óptimo para citación en AI Overviews es 134–167 palabras por respuesta. Respuestas tan breves no se citan como fragmentos autosuficientes.
Fix: Expandir cada respuesta del FAQ en el CMS (Firestore) a ~150 palabras, con detalle de procedimiento y contexto específico (ej: formato de folio JNS-XXXXXX, ventana de pago de 24 horas, confirmación por WhatsApp).
Medio /rifas-previas/[id] es "use client" — Google ve un shell vacío
La página fetcha datos en useEffect. El primer wave de crawl de Google ve contenido vacío, no puede inyectar schema, y no genera título dinámico.
Fix: Convertir a Server Component usando el mismo patrón de app/rifas/[id]/page.tsx. Usar generateStaticParams + generateMetadata con fetch desde Firebase Admin SDK.
Medio sitemap.ts usa new Date() — lastmod idéntico en todas las páginas
10 de 12 URLs en el sitemap muestran el mismo timestamp 2026-03-13T23:24:30.011Z. Google interpreta esto como fechas fabricadas e ignora el campo completamente, perdiendo la señal de frescura.
Fix en app/sitemap.ts: Reemplazar new Date() con fechas reales por ruta estática. El documento site_content/texts en Firestore ya tiene updatedAt — usarlo para las páginas de contenido CMS.
Medio Sin preconnect hints para Vercel Blob, Fonts y Clarity
Las imágenes de rifas se cargan desde Vercel Blob sin preconnect previo. El browser hace DNS + TCP + TLS antes del primer byte de imagen — ~100-200ms adicionales en LCP de /rifas.
Fix en app/layout.tsx: 
<link rel="preconnect" href="https://u0qlqi6owu46oklm.public.blob.vercel-storage.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
<link rel="preconnect" href="https://www.clarity.ms" />
Medio Sin apple-touch-icon, theme-color ni Web App Manifest
Usuarios de iOS que agregan el sitio al home screen ven un screenshot en vez del ícono de marca. Chrome en Android no colorea la barra de direcciones con el rojo de marca. El sitio no es instalable como PWA — relevante para clientes recurrentes.
Fix: Agregar en el metadata de app/layout.tsx: icons: { apple: "/apple-touch-icon.png" }, themeColor: "#e02020", y crear /public/manifest.json mínimo.
Medio HSTS en apex sin includeSubDomains ni preload
El subdominio www tiene HSTS completo (max-age=63072000; includeSubDomains; preload). El dominio apex (sorteosjans.com.mx) solo tiene max-age, sin los flags de subdominio ni preload. Bloquea el envío al HSTS Preload List.
Fix: Configurar headers en Vercel para que el apex también envíe Strict-Transport-Security: max-age=63072000; includeSubDomains; preload. Luego enviar en hstspreload.org.
Medio URLs de rifas exponen IDs de Firestore sin valor de keyword
Ejemplo: /rifas/bWLnZc44JHTgK96c84mi. La URL no aporta señal de keyword. Comparar con /rifas/iphone-15-pro-marzo-2026 que sí lo haría.
Fix (largo plazo): Agregar campo slug a los documentos de rifa en Firestore y usar en las rutas. Mantener compatibilidad por ID para rifas existentes con redirect.
